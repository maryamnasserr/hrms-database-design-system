# Human Resource Management System (HRMS) – Database & Web Application Project

## Overview
This project was developed as part of the Business Informatics program at the German International University of Applied Sciences. It presents a complete Human Resource Management System (HRMS) that integrates employee management, job assignments, performance appraisal, and training processes into a unified web-based system supported by a structured database design.

The system is designed to improve organizational efficiency by centralizing HR operations, ensuring data consistency, and enabling performance tracking and employee development.

## System Overview
The HRMS platform is built to manage the following core HR functions:
- Employee records and profiles
- Department and job structure management
- Job assignments and contracts
- Performance appraisal cycles and KPI tracking
- Training programs and certifications
- Employee appeals and performance reviews

---

## Application Pages

### Page 1 – Dashboard
The dashboard serves as the main entry point to the system. It provides:
- Quick navigation to all HR modules
- Key HR statistics including:
  - Total employees
  - Departments
  - Job positions
  - Training programs
  - Active job assignments
  - Performance cycles

---

### Page 2 – Employees
This page allows full employee management including:
- Adding new employees
- Searching employees by name
- Viewing employee profiles
- Editing and deleting employee records

Employee data includes:
- Personal and contact information
- Employment status (Active, Probation, etc.)
- Nationality and job-related details

---

### Page 3 – Departments
This module manages organizational departments:
- Add and update departments
- Search departments by name
- View department structure and details

Each department includes:
- Name, location, and email
- Type (Academic or Administrative)
- Available actions (edit/delete)

---

### Page 4 – Jobs
This page defines organizational roles:
- Create and manage job positions
- Search jobs by title or code
- View job details including:
  - Job code and title
  - Category and seniority level
  - Department assignment
  - Salary range
  - Job status

---

### Page 5 – Job Assignments
This module manages employee-job relationships:
- Assign employees to jobs
- Define contract type and salary
- Track start and end dates
- Monitor active and completed assignments

Each assignment includes:
- Employee and job details
- Contract type (full-time, part-time, probationary)
- Employment status (active/inactive)

---

### Page 6 – Training Programs
This module manages employee development programs:
- Add and manage training programs
- Search programs by title or code
- Track training type and delivery method

Each program includes:
- Code and title
- Type (professional, technical, soft skills, etc.)
- Delivery method (online, in-person, hybrid)
- Status (approved or pending)

---

### Page 7 – Performance Cycles
This module defines evaluation periods:
- Create and manage performance cycles
- View cycle details and duration
- Link cycles to employee appraisals

Includes:
- Cycle type (Annual, Quarterly, etc.)
- Start and end dates
- Submission deadlines

---

### Page 8 – Appraisals
This page manages employee performance evaluations:
- Create and edit appraisal records
- Link employees, jobs, and cycles
- Store performance scores and feedback

Includes:
- Overall performance score
- Manager, HR, and employee comments
- Reviewer details

---

### Page 9 – Appeals
This module handles performance disputes:
- Employees can submit appeals
- Track appeal status (Pending, Approved, Rejected)
- Store original and revised scores

Each appeal includes:
- Reason for appeal
- Submission date
- Outcome and updated evaluation

---

## Database Design Summary
The system is built on a relational database model that includes:

### Core Entities
- Employee
- Department
- Faculty
- Job
- Job Assignment
- Performance Cycle
- Appraisal
- KPI
- Training Program
- Appeal
- Contract

### Key Relationships
- Employees belong to departments and faculties
- Employees are assigned to jobs through job assignments
- Jobs contain objectives and KPIs
- Performance cycles evaluate employee performance
- Training programs track employee development

## Key Features
- Full CRUD operations for all HR entities
- Role-based HR process simulation
- Performance tracking using KPI-based evaluation
- Training lifecycle management
- Employee appeals and feedback system
